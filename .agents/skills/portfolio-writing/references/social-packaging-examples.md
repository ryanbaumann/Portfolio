# Social packaging few-shot examples

Use these as patterns, not templates. Preserve Ryan's voice and evidence standard.

## Field Note headline

Weak topic: `Developer experience metrics`
Better: `DevX Is Not a Docs Team`
Why: Names the misconception first, then opens a clear path to the mechanism: own friction, ship fixes, distribute them, and measure outcomes.

Weak topic: `Agent skills launch`
Better: `A Bigger Model Does Not Fix a Broken Developer Journey`
Why: Swaps the obvious launch framing for the underlying lesson. The product becomes evidence for the claim, not the headline.

Weak topic: `OAuth debugging notes`
Better: `The OAuth Bug Was Not in OAuth`
Why: Creates a specific contradiction that a technical reader wants resolved, without hiding the subject.

Weak topic: `Developer tool onboarding`
Better: `The First Trace Is the Onboarding Funnel`
Why: Leads with a mechanism, not a negation. The article can then show how a concrete trace reveals whether the first-run path works.

## Social draft

LinkedIn pattern:

```text
DevX teams often get pulled into publishing more docs.

That is usually the wrong scoreboard.

The better loop is: find repeated developer friction, ship the smallest fix, put the fixed path where builders already work, and measure whether behavior changed.

That changes DevX from an output function into a growth function.

<link>
```

X pattern:

```text
DevX is not a docs queue.

It is a loop: find friction, ship the fix, distribute the path, measure whether builders changed behavior.

<link>
```

## Social preview or thumbnail prompt pattern

Good prompt brief:

```text
One compact visual contrast: left side shows scattered docs and unresolved friction; right side shows a four-step loop labeled Friction, Ship, Distribute, Measure. High contrast, one focal point, no tiny text, no invented metrics, no logos, no exaggerated emotion.
```

Avoid:

```text
A generic title card with the article title repeated, many icons, lots of labels, and a dramatic reaction face.
```

## Naming

Weak demo name: `Map routing demo`
Better: `Commute Tradeoff Map`
Why: Names the decision the tool helps a builder make.

Weak talk title: `AI and developer experience`
Better: `When the First User Is an Agent`
Why: Puts the audience in the changed situation, then lets the abstract explain the mechanism.
